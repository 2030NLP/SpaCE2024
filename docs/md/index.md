### <center>1.  <span id="content">任务内容</span></center>

#### 1.1  任务简介

空间表达描述了物体之间的空间方位关系，是自然语言中的高频现象。要准确理解文本中空间表达的语义，不仅需要语言知识，还需要调用空间认知能力，构建空间场景，并基于世界知识进行空间方位信息相关的推理。

中文空间语义理解评测（Spatial Cognition Evaluation，简称 SpaCE）以测试机器的中文空间语义理解水平为目标，自 2021 年开始连续举办了三届赛事，发布了一系列评测任务，包括（1）空间信息正误判断；（2）空间信息异常识别；（3）空间信息要素分析与语义角色分类识别；（4）空间表达形义关系判别与异同原因解释，等等。现有评测结果显示，机器的中文空间语义理解水平与普通人类平均水平相比，在对空间认知加工要求较高的任务上，存在较大差距。空间语义理解对包括大语言模型在内的 NLP 系统来说，仍然是一项极具挑战性的任务。

跟前三届赛事的任务设计相比，**第四届中文空间语义理解评测（SpaCE2024）**更加注重针对**大语言模型**的空间语义理解能力的测试，宗旨是在**一个测试数据集**上考察机器中文空间语义理解的综合能力。为此，SpaCE2024 将不再划分子任务，以**选择题**的形式考察以下五个层次的空间语义理解能力：

（1）空间信息异常识别。要求从四个选项中选出空间信息异常的语言表达。

（2）空间信息实体识别。要求从四个选项中选出文本空间信息的参照物或目标物。

（3）空间信息角色识别。要求从四个选项中选出文本空间信息的语义角色，或者选出与所给语义角色相对应的空间表达形式。

（4）空间方位信息推理。要求基于空间知识进行推理，从四个选项中选出推理结果。

（5）空间形义关系判别。要求选择构成两个文本异形同义或异义的空间义词语。

**参赛队伍允许使用 ChatGPT 等大语言模型参赛，但必须提供所使用的调用代码和提示模板。**

#### 1.2  与SpaCE2023的比较

- 测试题形式改为选择题。尽量简化测试题形式，减少因形式复杂性对评测效度的影响。
- 测试题文本内容设置了一定比例的重复。目的是考察语言模型表现的一致性和稳定性。重复题包括两种情况：a.文本内容完全相同；b.内容相同但有形式差异。
- 测试题文本来源于专业领域的语料占比提升。SpaCE2024更加注重测试题答案的客观性，减少了文学类语料的占比，提高了交通事故现场描述文本、人体体育动作文本、地理百科文本等专业领域语料的占比。

### <center>2.  <span id="data">评测数据</span></center>

#### 2.1  <span id="ziduan">数据字段</span>

（1）text：文本材料。机器需要阅读理解 text 后回答问题。

（2）question：机器需要回答的问题。

（3）option：选择题的选项。一个数组，每个元素是一个选项。

（4）answer：选择题的答案。一个数组，每个元素是目标选项在option数组的下标（从0开始），如[0]表示机器选择option的第一个选项作为答案。

#### 2.2  <span id="example">数据样例</span>

各能力层次的题目样例及说明如下。

##### 2.1.1  <span id="task1">空间信息异常识别</span>

输入：

```json
{"text":"孟某驾驶一辆未悬挂法定号牌的电动自行车从小区西门出发，后座载着同事何某。驶出西门后，孟某右转沿凉城路由南向北行驶至车站南路路口附近，随后右转进入车站南路。此时，谈某正驾驶电动自行车在前方同向骑行，孟某从右侧超越谈某的过程中，孟某的电动自行车右侧与谈某的电动自行车左侧发生碰擦，致谈某向右倒地。孟某加速往东逃逸，后被前方水电路路口的交警拦停抓获。", 
 "question":"哪一个选项是text中的异常空间信息？", 
 "option":["孟某右转进入车站南路",
           "孟某从右侧超越谈某",
           "孟某加速往东逃逸",
           "以上选项都不是"]}
```

输出：

```json
{"answer":[1]}
```

说明：

- 空间信息异常识别题的 text 文本中含有**异常**空间信息。例题中的异常空间信息是“孟某从**右侧**超越谈某的过程中，孟某的电动自行车右侧与谈某的电动自行车左侧发生碰擦”。根据空间方位常识可知，孟某如果从右侧超越谈某，只能是孟某的左侧与谈某的右侧发生碰擦，不可能孟某的右侧与谈某的左侧发生碰擦。
- question 要求识别出 text 中异常空间信息所在的具体文本片段。
- option 包含四个选项，其中三个选项是 text 中的空间信息。最后一个选项是“以上选项都不是”，表示以上三个选项都不能回答 question。
- answer 是答案。例题的正确答案是 option 中下标为 1 的选项“孟某从右侧超越谈某”，输出 [1]。

##### 2.1.2  <span id="task2">空间信息实体识别</span>

输入：

```json
{"text":"孟某驾驶一辆未悬挂法定号牌的电动自行车从小区西门出发，后座载着同事何某。驶出西门后，孟某右转沿凉城路由南向北行驶至车站南路路口附近，随后右转进入车站南路。此时，谈某正驾驶电动自行车在前方同向骑行，孟某从左侧超越谈某的过程中，孟某的电动自行车右侧与谈某的电动自行车左侧发生碰擦，致谈某向右倒地。孟某加速往东逃逸，后被前方水电路路口的交警拦停抓获。", 
 "question":"“谈某驾驶电动自行车在前方同向骑行”指的是“谈某”在哪一个选项的“前方”？", 
 "option":["水电路",
           "交警",
           "孟某",
           "以上选项都不是"]}
```

输出：

```json
{"answer":[2]}
```

说明：

- 空间信息实体识别题的 text 是一个空间信息**正常**的文本。相比 2.1.1 的异常文本，例题的文本是“孟某从**左侧**超越谈某的过程中，孟某的电动自行车右侧与谈某的电动自行车左侧发生碰擦”，没有异常。
- question 要求针对 text 中的某个空间方位信息表述，识别该空间方位信息相关的空间实体（如参照物或目标物）是什么。
- option 包含四个选项，其中三个选项是 text 中的实体。最后一个选项是“以上选项都不是”，表示以上三个选项都不能回答 question。
- answer 是答案。例题中“谈某……在前方同向行驶”，指的是谈某在孟某之前，而相应地，孟某则是在谈某的后方骑行。所以输出 [2]，代表 option 中下标为 2 的选项“孟某”。

##### 2.1.4  <span id="task4">空间方位信息推理</span>

输入：

```json
{"text":"孟某驾驶一辆未悬挂法定号牌的电动自行车从小区西门出发，后座载着同事何某。驶出西门后，孟某右转沿凉城路由南向北行驶至车站南路路口附近，随后右转进入车站南路。此时，谈某正驾驶电动自行车在前方同向骑行，孟某从左侧超越谈某的过程中，孟某的电动自行车右侧与谈某的电动自行车左侧发生碰擦，致谈某向右倒地。孟某加速往东逃逸，后被前方水电路路口的交警拦停抓获。", 
 "question":"“右转进入车站南路”后，哪一个选项是“孟某”的移动方向？", 
 "option":["往北",
           "往西",
           "往南",
           "以上选项都不是"]}
```

输出：

```json
{"answer":[3]}
```

说明：

- 空间方位推理题的 text 是一个空间信息正常的文本。例题与 2.1.2 使用同一个 text 。
- question 要求根据 text 已有的空间方位信息表述，推理空间实体未在 text 中写明的空间方位信息。
- option 包含四个选项，其中三个选项是推理得出的空间方位信息。最后一个选项是“以上选项都不是”，表示以上三个选项都不能回答 question。
- answer 是答案。例题中，孟某的初始移动方向是“由南向北”，说明初始移动（未转向之前）时孟某的朝向是北方。根据空间方位常识可知，孟某右转后将“往东”移动。option 中没有给出“往东”的选项，所以输出 [3]，代表 option 中下标为 3 的选项“以上选项都不是”。

##### 2.1.5  <span id="task5">空间形义关系判别</span>

输入：

```json
{"text":"贵州山路多，山路多有长下坡。长途货车司机在长下坡路段行驶时，需要注意车辆的制动，否则容易因刹车片过热导致车辆不受控制向前方滑行，引发交通事故。", 
 "question":"哪一个选项替换“向前方滑行”中的“前方”，能构造出与text语义相同且符合常识的文本？", 
 "option":["上方",
           "下方",
           "后方",
           "以上选项都不是"]}
```

输出：

```json
{"answer":[1]}
```

说明：

- 空间形义关系判别题的 text 是一个空间信息正常的文本。
- question 要求用选项去替换 text 中的目标词，替换后的文本符合常识，而且与 text 的语义相同或不同。
- option 包含四个选项，其中三个选项是替换词。最后一个选项是“以上选项都不是”，表示以上三个选项都不能回答 question。
- answer 是答案。例题中，在“车辆在下坡路段行驶”的语境下，“向前方滑行”和“向下方滑行”都描述了车辆沿着坡道向下行驶的场景。所以输出 [1]，代表 option 中下标为 1 的选项“下方”。
